```
POST /api/auth/organization/update-member-role
```

Updates the role(s) assigned to an existing organization member.

<Callout>This endpoint uses **POST**, not PUT or PATCH.</Callout>

<Callout type="warn">This endpoint authenticates with your logged-in dashboard **session**, not an API key. Send your session cookie with the request.</Callout>

### Request Body

<TypeTable
  type={{
    role: {
      description: 'The new role(s) to assign. One of `owner`, `admin`, `member` (or an array of roles).',
      type: 'string | string[]',
      required: true,
    },
    memberId: {
      description: 'The ID of the member to update',
      type: 'string',
      required: true,
    },
    organizationId: {
      description: 'The organization the member belongs to. Defaults to your active organization.',
      type: 'string',
      required: false,
    },
  }}
/>

### Response

Returns the updated member object directly — the fields are at the top level, **not** nested under a `member` key.

<TypeTable
  type={{
    id: {
      description: "The member record's identifier",
      type: 'string',
    },
    userId: {
      description: "The underlying user's ID",
      type: 'string',
    },
    organizationId: {
      description: 'The organization the member belongs to',
      type: 'string',
    },
    role: {
      description: "The member's updated role",
      type: 'string',
    },
    createdAt: {
      description: 'When the membership was created',
      type: 'string',
    },
  }}
/>
